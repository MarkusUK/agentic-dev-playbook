# UI Assets Skill

Use this skill when sourcing or adapting components, icons, animations, motion, or UI inspiration.

## Rules

1. Treat external UI/component output as inspiration, not final code.
2. Adapt to the project's existing design system.
3. Avoid adding dependencies unless necessary.
4. Check licences before copying substantial component code or assets.
5. Prefer accessible, responsive, lightweight UI.
6. For animations, respect reduced-motion preferences where applicable.
7. For Rive assets, keep `.riv` files in the agreed asset directory and document state machine inputs.

## Output

```text
Asset/component source:
- ...

How it was adapted:
- ...

Dependencies added:
- ...

Licence/risk notes:
- ...

- ...
```
