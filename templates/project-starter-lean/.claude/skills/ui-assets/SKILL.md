# UI Assets Skill

Use this skill when sourcing or adapting visual assets, UI components, icons, animation, or design inspiration.

## Rules

1. Treat external UI/component output as inspiration, not final code.
2. Adapt everything to the project's design system.
3. Avoid dependencies unless necessary.
4. Check licences before copying substantial code or assets.
5. Respect reduced-motion preferences for animation.
6. Document any asset source, dependency, or licence risk.

## Output

```text
Asset/component source:
How it was adapted:
Dependencies added:
Licence/risk notes:
Tests/build:
```
