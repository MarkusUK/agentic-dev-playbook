# Implementation Handoff

Use this when the design/spec is approved and ready for Codex.

## Summary

```text
What should be built?
Why is it useful?
What is intentionally out of scope?
```

## Approved Design Inputs

```text
Product brief:
Brand kit:
Screen spec:
Mockup/Figma/screenshot links:
```

## Implementation Plan

Ask Codex to propose this before editing:

```text
Read AGENTS.md, docs/architecture.md, docs/testing.md, and this handoff.

Do not edit yet.

Propose:
- files likely to change
- existing components/patterns to reuse
- risks
- tests/checks to run
- smallest safe implementation plan
```

## Definition of Done

- Screen matches approved spec closely.
- Existing design system/components are reused where possible.
- Responsive layout is checked.
- Loading, empty, error, and success states are handled.
- Relevant tests/checks pass or failures are explained.
- Claude Code reviews the diff before merge.
