# 05 - Screen Specs

Use this to turn a chosen brand direction into practical screen plans.

## Copy/paste prompt

```text
Using the selected design direction below:

[Paste selected direction / brand kit]

Generate implementation-ready screen specs for:

1. Onboarding
2. Home/dashboard
3. Main detail screen
4. Create/edit flow
5. Reflection/history/list screen
6. Settings/profile
7. Empty state
8. Error state
9. Loading state

For each screen include:

- user goal
- layout description
- component list
- content hierarchy
- states
- interactions
- accessibility notes
- design tokens used
- reusable components
- platform-specific notes for [Android/iOS/web/etc.]

Do not write production code yet.
```

## Output expectation

You want a spec that can later be handed to Codex or Claude for implementation.

Ask for:

```text
- tokens
- components
- states
- interactions
- accessibility
- platform notes
```

Not just “make it pretty”.
