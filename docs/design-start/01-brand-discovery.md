# 01 - Brand Discovery

Use this before asking for screens or code.

## Copy/paste prompt

```text
I want to design a new app/product.

Project idea:
[Describe the app in 2-5 sentences]

Target users:
[Who is this for?]

Primary goal:
[What should users accomplish?]

Theme / brand direction:
[Example: Japanese sakura theme, calm, elegant, seasonal, premium but approachable]

Platform:
[Android / iOS / web / desktop / cross-platform]

Tech stack if known:
[Example: Android Jetpack Compose / React Native / React + Tailwind / Flutter]

Design constraints:
- Must be accessible
- Must support light and dark mode
- Must be buildable with the intended tech stack
- Should avoid generic AI-looking UI
- Should not add unnecessary dependencies
- Should feel coherent as a brand system, not just a pretty screen

Available tools:
- Figma MCP: [yes/no]
- Google Stitch MCP: [yes/no]
- 21st.dev/Magic MCP: [yes/no]
- Rive: [yes/no]
- Existing design system: [yes/no/details]

Task:

Do not write code yet.

Act as a product designer, brand strategist, and senior app designer.

First, produce a design discovery brief with:

1. Product summary
2. Target audience
3. User emotions we want to create
4. Brand personality
5. Visual principles
6. Accessibility considerations
7. Design risks or clichés to avoid
8. Visual references or metaphors
9. Platform-specific design considerations
10. What not to do

Then ask me which direction I want to explore next.
```
